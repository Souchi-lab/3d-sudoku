# Hyper‑Cube Sudoku 仕様書 v0.9

> **目的** ― 可変サイズ 3D 数独をベースに、タイムアタック・オンラインランキング・完全ランダマイズ盤面を備えた "Hyper‑Cube Sudoku" を実装する。Gemini CLI へそのまま渡し、段階的に開発タスクへ分解できる粒度で記述する。

---

## 0. 用語

| 用語            | 説明                               |
| ------------- | -------------------------------- |
| **N**         | 立方体一辺のセル数 (3 ≤ N ≤ 5)            |
| **層 (Slice)** | X/Y/Z いずれかの固定座標で切り取った N×N の 2D 面 |
| **ライン**       | 行・列・柱 (X 行, Y 列, Z 柱) の総称        |
| **ヒントセル**     | ゲーム開始時に固定配置される編集不可セル             |
| **TA モード**    | Time‑Attack モード                  |

---

## 1. ゲームモード

| モード              | 説明                                     |
| ---------------- | -------------------------------------- |
| **スタンダード**       | 時間無制限。全セルを正しく埋めればクリア。                  |
| **タイムアタック (TA)** | 残り時間が 0 になる前に全セルを埋めるとクリア。残り時間は行動で増減する。 |

※ MVP ではソロプレイのみ。オンライン対戦は今後フェーズ追加。

---

## 2. ルール

1. 使用数字は **1 〜 N**。
2. 同じ数字は **X 行 / Y 列 / Z 柱** 内で重複してはならない。
3. **サブキューブ制約なし** (アルゴリズムと UI を簡素化)。
4. `GameState.sequence` で生成された乱序列から **1 手に 1 個** の数字を供給し、それを合法マスへ配置する。
5. **ランダマイズ 3 レイヤー** を毎ゲーム適用して盤面の暗記を防止。詳細は §6 を参照。
6. **置ける合法マスが一つも無い場合**

   * **TA モード**: プレイヤーは **「強制スキップ」** (残り時間 −30 s / −20 pt) を **1 ゲームにつき 1 回だけ** 使用可能。それ以降はスキップ不可でゲームオーバー。
   * **スタンダード**: 強制スキップは −20 pt (回数制限なし)。合法マスが無くなり、かつ残りスキップ回数が 0 の場合は失敗扱い。

## 6. 盤面スクランブル API

```python
# scramble.py  (Utility)

def scramble_board(base_board: list[list[list[int]]], N: int, seed: int | None = None):
    """完成盤テンプレ (N×N×N) を 3‑Layer ランダマイズして返す。"""
    import random; rnd = random.Random(seed)
    # ① 数字置換
    perm = list(range(1, N + 1)); rnd.shuffle(perm)
    repl = {i + 1: perm[i] for i in range(N)}
    # ② 軸シャッフル + 反転
    axes = ['x', 'y', 'z']; rnd.shuffle(axes)
    flips = [rnd.choice([1, -1]) for _ in range(3)]
    # ③ 層スライド
    shifts = [rnd.randrange(N) for _ in range(3)]
    def map_pos(i, j, k):
        v = {'x': i, 'y': j, 'z': k}
        a, b, c = v[axes[0]], v[axes[1]], v[axes[2]]
        a = (flips[0] * (a + shifts[0])) % N
        b = (flips[1] * (b + shifts[1])) % N
        c = (flips[2] * (c + shifts[2])) % N
        return a, b, c
    board = [[[None] * N for _ in range(N)] for _ in range(N)]
    for i in range(N):
        for j in range(N):
            for k in range(N):
                x, y, z = map_pos(i, j, k)
                board[x][y][z] = repl[base_board[i][j][k]]
    return board
```

## 10. Gemini CLI への依頼タスクリスト例

1. **Constants & GameState**: N 可変対応 + タイマー実装。
2. **scramble.py**: §6 のユーティリティ実装。
3. **Field.py**: 行/列/柱/層 完成判定 + ボーナス発火。
4. **cube3d\_view\.py**: レベル依存 UI (ズーム Fit, 透過, タイマー表示)。
5. **main.py**: レベルセレクト画面 + ヒント配置 + スコア/タイム UI 更新。
6. **API**: Flask ミニサーバで `/api/rank` 実装 (SQLite)。
7. **テスト**
8. **ユニットテスト (pytest)**

   * 対象 : **N ∈ {3, 4, 5}**
   * 完成盤テンプレを `scramble_board()` で 100 ケースずつ生成し、`Field.check()` / `Field.reflect()` / `GameState.create_nums()` の整合性を検証。
   * 行・列・柱・層完成時にボーナスが正しく発火することをアサート。
   * タイマー減算／ボーナス加算の境界値（0 s 付近）をテスト。
9. **自動プレイボット `ai_autoplay.py`**

   * アルゴリズム : 候補数が最小のセルを優先し、詰まったら「強制スキップ」発動。
   * N=3〜5 を各 10 回実行し、平均クリアタイム・平均スコアを CSV で出力。
   * 合法マス枯渇→ゲームオーバー／スキップ上限超過のハンドリングを回帰テスト。
10. **CI 連携 (GitHub Actions)**

    * ワークフロー `ci.yml` で `pytest -n auto` と `python ai_autoplay.py --ci` を実行。
    * 成功基準 : 全ユニットテスト通過 & 10/10 回完走 (スキップ回数 ≤ 1)。
11. **ベンチマークレポート自動生成**

    * `scripts/gen_report.py` で CSV → Markdown (`/docs/benchmarks/latest.md`) 生成。
